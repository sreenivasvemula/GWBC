Conversion of Web Services from RPCE to WS-I

The following web services were converted from RPCE to WS-I web services in version 8.0.1. WS-I is a newer format of web services. See “Web Services Introduction” in the Integration Guide.
IMPORTANT RPCE web services are deprecated. Convert existing RPCE code to WS-I web services.
One important difference in the conversion is the handling of entity instance data. Because WS-I web services do not support entity data directly as method arguments or return values, many APIs now use a separate data transfer object (DTO) to encapsulate entity data. For example, instead of passing an Address entity directly, it might pass a Gosu class (the DTO) called AddressDTO. The DTO class contains only the properties in an Address entity instance that are necessary for the web service. For more information about DTOs in general, see “BillingCenter Data Transfer Objects (DTOs)” in the Integration Guide.
If you extend the entity data model with properties that must exist in the web service for sending or receiving data, you must also extend the data transfer object. Add to the set of properties in the corresponding Gosu class with the DTO suffix. For example, if you add an important property to the Address entity, also add the property to the Gosu class AddressDTO.
WARNING Before editing Gosu class DTO files, carefully review the introductory comments at the top of each DTO file.

IBillingCenterAPI
gw.webservice.policycenter.bc800.BillingAPI
Billing instructions are now handled in the BillingAPI WS-I web service. See “Policy Administration System Core Web Service APIs (BillingAPI)” in the Integration Guide
 
gw.webservice.bc.bc801.InvoiceDetailsAPI
For methods related to invoice details, see the InvoiceDetailsAPI WS-I web service. See “Invoice Details Web Service APIs (InvoiceDetailsAPI)” in the Integration Guide.
 
gw.webservice.bc.bc801.BCAPI
For most methods unrelated to billing instructions or invoice details, see the BCAPI WS-I web service. See “Policy Administration System Core Web Service APIs (BillingAPI)” in the Integration Guide.

For web service methods related to collateral, see the BCAPI WS-I web service. See “Other Billing Web Service APIs (BCAPI)” in the Integration Guide
IPolicyPeriodAPI
gw.webservice.bc.bc801.BCAPI
For web service methods related to changing billing methods, see the BCAPI WS-I web service. See “Other Billing Web Service APIs (BCAPI)” in the Integration Guide.
IInvoiceAPI
gw.webservice.bc.bc801.BCAPI
There was only one method in the RPCE web service IInvoiceAPI. The method previewed an installment schedule. As of version 8.0.1, for this task use the BillingAPI WS-I web service methods previewInstallmentPlanInvoices and previewInstallmentsPlanInvoices. See “Other Billing Web Service APIs (BCAPI)” in the Integration Guide.
On a related note, there was a WS-I web service InvoiceAPI that contained the same methods as the RPCE web service IInvoiceAPI. The InvoiceAPI web service is now deprecated. Use the BillingAPI methods instead.
IStagingTableEncryptionAPI
gw.webservice.bc.bc801.BCAPI
or gw.wsi.pl.TableImportAPI
There were only two methods in the RPCE web service IStagingTableEncryptionAPI. The methods duplicate the staging table encryption methods in the table import web service. In this release, you can find these methods in both the BCAPI and TableImportAPI web services. They do the same task, although the BCAPI version of the method encryptDataOnStagingTables does not return the result value. See “Other Billing Web Service APIs (BCAPI)” in the Integration Guide.
