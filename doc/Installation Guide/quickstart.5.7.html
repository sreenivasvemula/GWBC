Installing Sample Data

This topic explains how to load the sample data included with the base BillingCenter installation, and describes how the sample data loading mechanism can be reconfigured.
BillingCenter includes sample data for use in training, configuration and testing. Guidewire strongly recommends that you not load sample data into a production system.
The BillingCenter server must be in development mode to load sample data. The QuickStart server is always in development mode. If you are using a Tomcat server in your development environment, start the server in development mode. See “Building a BillingCenter Development Environment with Tomcat”.
If you have already loaded a sample data set, you must drop the database before you load a different sample data set.
To install sample data
1.
Log into BillingCenter with the su account.
2.
Press ALT+SHIFT+T to open the Server Tools page.
3.
Click Internal Tools.
4.
Click BC Sample Data in the menu on the left.
5.
Click Import.
BillingCenter now contains sample data.

Loading Sample Data Using Gosu Classes

The Import Sample Data button in BillingCenter causes sample data to be imported using a Gosu class called SampleData.DataSets. The SampleData.DataSets class is configured at BillingCenter startup, when BillingCenter loads config.xml. The config.xml file contains a parameter that BillingCenter uses to determine how to load sample data.
To configure the way BillingCenter loads sample data
1.
In Studio, select configuration → Classes → Sample Data → DataSets to open DataSets.gs.
Note: To open DataSets.gs quickly in Studio, type CTRL+N and enter DataSets.
2.
In DataSets.gs, use the createBillingCenterSampleData function as an example for creating a new method to make the sample data. Pass dataGenerator as the first parameter.
3.
In the GosuSampleDataMethod node of config.xml, replace createBillingCenterSampleData with the name of your new method.
4.
Save changes in Studio. Clicking the Import Sample Data button now invokes your new method.
