Account Evaluation Customization

As mentioned earlier, the exact algorithm by which the system generates an account evaluation rating is customer-definable by implementing a plugin interface. BillingCenter includes an internal reference implementation for this plugin interface if the customer does not define it. The reference implementation may be suitable for some customers, but it can be customized by implementing this plugin interface in Gosu or Java. In addition to the algorithm itself, you can use this plugin to customize the number of days to search for delinquencies, cancellations, and pejorative payment reversals. This includes insufficient funds / NSFs.
The reference implementation for this plugin defines the time period to search for adverse events to be 365 days in the past and implements the following rating algorithm:
Account Evaluation
Conditions
New Account
Any account opened within the last 365 days
Excellent
Up to 1 Delinquency for the rolling 12 month time period and 0 NSFs and 0 Policy Cancellations
Good
Up to 2 Delinquencies and 0 NSFs and 1 Policy Cancellation
Acceptable
Up to 4 Delinquencies and 1 NSFs and 2 Policy Cancellations
Marginal
Up to 6 Delinquencies and 2 NSFs and 3 Policy Cancellations
Poor
Anything else
For more details of the IAccountEvaluationCalculator plugin, refer to “Account Evaluation Calculation Plugin” in the Integration Guide.
If you wish to different rating values other than the built-in ones (Excellent, Good, and so on), you can extend them in the AccountEvaluation typelist defined in the tl_bc_account.xml file. Its reference implementation includes these values:
<typelist name="AccountEvaluation" desc="Evaluation rating for an account">
   <typecode code="newaccount" name="New Account" desc="New Account"/>
   <typecode code="excellent" name="Excellent" desc="Excellent"/>
   <typecode code="good" name="Good" desc="Good"/>
   <typecode code="acceptable" name="Acceptable" desc="Acceptable"/>
   <typecode code="marginal" name="Marginal" desc="Marginal"/>
   <typecode code="poor" name="Poor" desc="Poor"/>
 </typelist>
