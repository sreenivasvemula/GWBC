About String Resources

BillingCenter uses string resources for the following:
•
Display Keys – Strings to display as field and screen labels and interactive error messages
•
Typecodes – Strings to display as choices in drop-down lists
•
Workflow Step Names – Strings to display as individual step names in workflow processes
•
Document Assistant Messages – Strings to display as error and informational messages by the Document Assistant
•
Script Parameter Descriptions – Strings to display as descriptions of script parameters
You can extract these string resources from BillingCenter and localize them separately from other application resources.

Display Keys

BillingCenter stores as key/value pairs the United States English string resources from which it generates field and screen labels and interactive error messages in the user interface. Guidewire calls these particular key/value pairs display keys. You specify the key/value pair of a display key in standard Java properties syntax. For example:
Admin.Workload.WorkloadClassification.General = General
BillingCenter stores the key/value pairs for display keys in a file called display.properties. In the base configuration, BillingCenter contains a single copy of display.properties, with string resources in United States English. In Guidewire Studio, you can navigate to this display.properties file in the Project window as follows:
configuration → config → Localizations → en_US
If you install a language pack for a language other than U.S. English, the installer adds a version of display.properties localized for that language.

Typecodes

BillingCenter stores as key/value pairs the U.S. English string resources from which it displays choices and choice descriptions in drop-down lists in the user interface. Guidewire calls these particular key/value pairs typecodes. You specify the key/value pairs for the name and description of a typecode in standard Java properties syntax. For example:
TypeKey.CoverageType.CPBldgCov = Building Coverage
TypeKeyDescription.CoverageType.CPBldgCov = Building Coverage
BillingCenter stores the key/value pairs for typecodes in a file called typelist.properties. In the base configuration, BillingCenter contains a single copy of typelist.properties, with string resources in U.S. English. In Guidewire Studio, you can navigate to the typelist.properties file in the Project window, as follows:
configuration → config → Localizations → en_US
If you install a language pack for a language other than U.S. English, the installer adds a version of typelist.properties localized for that language.

Script Parameter Descriptions

BillingCenter displays script parameter descriptions in the detail view of each script parameter. Navigate to Administration tab → Utilities → Script Parameters and click the name of a parameter to see its detail view. You can localize a script parameter description by adding a key to the display.properties file.

Localizing Document Assistant Messages

The Document Assistant is a Java applet that supports client-side creating, viewing, editing, and uploading of files. This applet loads its error messages from display keys in the display.properties file. To localize Document Assistant messages, look for display keys in display.properties that start with Document.Assistant.
