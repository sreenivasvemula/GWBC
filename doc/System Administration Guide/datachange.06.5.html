Data Change Web Service Reference (DataChangeAPI)

To register a data change, or to check status on it, use methods on the DataChangeAPI web service.

DataChangeAPI method
Description
updateDataChangeGosu
Register a data change. Pass the method the reference ID, a human-readable description, and the Gosu code to run. Pass all of these arguments as String objects. The method returns the public ID of the new DataChange entity instance.
If the data change succeeded with no compile errors, you cannot edit it. You must re-register the script with a new reference ID.
If the data change was never run, or had compile errors, you can update (edit) the Gosu code with the same reference ID.
discardDataChange
This option indicates you want to discard a data change that you already registered. Pass a data change reference ID as a String. You cannot discard a data change that was already run.
getDataChangeResult
This option indicates you want the result of a data change that you already registered. Pass a data change reference ID. It returns a String that represents the results in the DataChange entity instance. If a user attempted to run it and there were parse errors, the results include the errors.
getDataChangeStatus
This option indicates you want the status of a data change that you already registered. Pass a data change reference ID. The method returns a DataChangeStatus typecode. Values include Open, Discarded, Executing, Failed, Completed.
