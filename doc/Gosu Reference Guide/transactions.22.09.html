Removing Entity Instances from the Database

Mark an entity for removal from the database by using the Bundle interface’s delete method.
For example:
uses gw.transaction.Transaction
 
var a123 = Transaction.getCurrent().loadBean(new Key(Address, 123))
 
Transaction.getCurrent().delete(a123) // add “instruction” to bundle to delete a123 from the database
When the bundle is committed, the entity is deleted from the database. However, if the entity’s data model configuration declares the entity as Retireable then the entity is not deleted, but is retired instead. For information about the Retireable entity type, see “Data Entities and the Application Database” in the Configuration Guide.
WARNING An entity can include foreign key links to other entities. In addition, other entities can be linked to the deleted entity. The delete method removes only the specified entity from the database; it does not remove entities linked to it. This situation can compromise data integrity. Additional situations exist where the removal of an entity can result in compromised data. For these reasons, it is strongly recommended to never delete entities from the database.
